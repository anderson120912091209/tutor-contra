## 🎯 底部导航（参考图片）

### 布局结构
```
┌─────────────────────────────────────┐
│  [上一步]     • ━ ━      [下一步]    │
│               1 / 3                 │
└─────────────────────────────────────┘
```

### 页面指示器动画
```typescript
{STEPS.map((step) => (
  <button
    className={`
      w-2 h-2 rounded-full transition-all
      ${currentStep === step.id 
        ? "bg-primary w-8"        // 当前页：加长
        : currentStep > step.id
        ? "bg-primary/60"         // 已完成：半透明
        : "bg-gray-300"           // 未完成：灰色
      }
    `}
  />
))}
```

### 视觉状态
- **当前页**：实心蓝色长条（8px 宽）
- **已完成页**：实心蓝色圆点（60% 透明度）
- **未完成页**：灰色圆点

## 🎨 颜色方案

### 主色调
```css
/* 进度徽章 */
bg-red-100 text-red-600  /* 16% */

/* 进度条 */
bg-red-500               /* 填充部分 */
bg-gray-200              /* 未填充部分 */

/* 建议提示 */
text-green-600           /* +84% 新增XXX */

/* 按钮 */
bg-blue-600              /* 主要按钮 */
```

### 为什么选择红色进度条？
参考图片使用**红色**来吸引注意力，表示"还有很多需要完成"。当进度提升，用户会有更强的成就感。

## 🎯 智能提示系统

### 动态建议
```typescript
const getNextStepSuggestion = () => {
  if (!displayName) return "新增顯示名稱";
  if (!bio) return "新增自我介紹";
  if (subjects.length === 0) return "新增教學科目";
  if (!location) return "新增教學地點";
  if (!yearsExperience) return "新增教學年資";
  if (education.length === 0) return "新增學歷背景";
  return "完善您的檔案";
};
```

### 显示逻辑
```
进度 < 100% → 显示建议
进度 = 100% → 隐藏建议
```

### 示例
```
16% → +84% 新增顯示名稱
28% → +72% 新增自我介紹
42% → +58% 新增教學科目
70% → +30% 新增學歷背景
100% → （不显示）
```

## 💡 设计理念

### 1. **渐进式引导**
- 不是一次性展示所有表单
- 一步步引导用户完成
- 降低认知负担

### 2. **即时反馈**
- 输入即更新预览
- 进度条实时变化
- 建议动态调整

### 3. **清晰的进度感知**
- 百分比数字
- 可视化进度条
- 还差多少的提示
- 页面指示器

### 4. **灵活导航**
- 可以前进/后退
- 可以跳转到任意步骤
- 不强制线性流程

## 📊 效果对比

### 用户完成率预期

**单页表单（旧）：**
- 看到所有表单 → 感到overwhelmed
- 不知道进度 → 失去动力
- 完成率：~40%

**多步骤表单（新）：**
- 一次看一个步骤 → 感觉简单
- 清楚知道进度 → 有成就感
- 完成率：~70%+ （预期提升）

### 用户反馈（预期）
- ✅ "很清楚知道还要填什么"
- ✅ "看到进度条增长很有动力"
- ✅ "不会觉得表单很长很复杂"
- ✅ "随时可以预览效果，很方便"

## 🎨 美学原则

### 1. **留白**
- 表单不拥挤
- 视觉呼吸空间
- 专注于当前任务

### 2. **对齐**
- 左侧：表单内容
- 右侧：预览内容
- 底部：导航居中对齐

### 3. **层次**
- 进度 > 标题 > 表单 > 提示
- 主按钮 > 次要按钮
- 当前步骤 > 其他步骤

### 4. **一致性**
- 间距系统统一
- 颜色使用一致
- 交互模式相同
